# 态极共振场架构（Resonance Field Architecture）

> **核心思想**：态极不是一个大的 Transformer 模型，而是由若干独立的小模型神经元组成。
> 所有神经元通过一个共享的"共振场"协同工作，从中涌现出统一的智能。
> 自我进化通过增加/替换神经元实现，递归通过场自然淘汰不共振的神经元实现。

---

## 1. 为什么不是 MoE

MoE（Mixture of Experts）的核心是"分而治之"——路由器选择 top-k 个专家来处理输入。

| | MoE 路由 | 共振场 |
|---|---|---|
| 神经元激活 | 路由器决定选谁 | **所有神经元同时激活，场自然筛选** |
| 协同方式 | 加权投票（线性） | **循环共振（乘性交互）** |
| 1+1 | 最多等于 2 | **可以大于 2** |
| 规模增长 | 增加专家 → 重新训练路由器 | 增加神经元 → 即插即用 |
| 进化淘汰 | 需要外部评估 | **场自然排斥不共振的神经元** |

共振场的核心区别：不是"选几个用"，而是"所有人同时说话，场决定听谁的"。

## 2. 整体架构

```
                        输入
                          |
                ┌─────────┼─────────┐
                |         |         |
           ┌────▼───┐ ┌───▼────┐ ┌───▼────┐
           |神经元 A | |神经元 B| |神经元 C|   ← 全量激活（所有 52 个）
           | 50M 参数| | 50M 参数| | 50M 参数|
           | hidden | | hidden | | hidden |
           └────┬───┘ └───┬────┘ └───┬────┘
                |         |         |
                └─────────┼─────────┘
                          |
                   ┌──────▼──────┐
                   |   共振场     |  ← D=4096 维共享矩阵
                   |  (Field)    |     所有神经元写入
                   └──────┬──────┘     所有神经元读回
                          |
                ┌─────────┼─────────┐
                | 读回     | 读回     | 读回
           ┌────▼───┐ ┌───▼────┐ ┌───▼────┐
           |神经元 A | |神经元 B| |神经元 C|   ← 第二轮循环
           | 条件化  | | 条件化 | | 条件化 |   被场状态影响后的重新推理
           └────┬───┘ └───┬────┘ └───┬────┘
                |         |         |
                └─────────┼─────────┘
                          |
                   ┌──────▼──────┐
                   |  更新共振场  |
                   └──────┬──────┘
                          |
                   循环 3-5 轮
                          |
                   ┌──────▼──────┐
                   |   输出       |
                   └─────────────┘
```

**关键特征**：
- 所有神经元同时激活（不存在"选谁不选谁"）
- 每轮循环：神经元写入场 → 从场读回 → 重新计算
- 场的介入让神经元的输出彼此"条件化"
- 多轮循环后，场的干涉模式自然涌现为输出

## 3. 神经元规格设计

神经元不是一样大的。三种规格，三种角色：

### 3.1 规格表

| | 紧凑型 | 标准型 | 专家型 |
|---|---|---|---|
| **参数量** | 15-25M | 40-60M | 80-120M |
| **hidden_size** | 512 | 768 | 1024 |
| **num_layers** | 6 | 10 | 14 |
| **num_heads** | 8 | 12 | 16 |
| **num_kv_heads** | 2 | 4 | 4 |
| **intermediate_size** | 1536 | 2304 | 3072 |
| **4090D 训练时间** | 3-5 分钟 | 10-15 分钟 | 25-35 分钟 |
| **角色** | 快速试错、探索新领域 | 主力神经元 | 复杂推理、代码、数学 |
| **生命周期** | 短（容易淘汰） | 中 | 长（通过进化验证） |
| **推荐数量** | 20-40 个 | 8-15 个 | 2-4 个 |

### 3.2 设计原理

**太小不行**：<5M 的模型连"理解一句话"都做不到，加到共振场里只是噪音。

**太大不行**：>200M 训练慢、进化慢，丧失了即插即用的灵活性。而且过大的神经元会对场形成"主导效应"，压制小神经元，破坏共振。

**异质化的意义**：用态极现有 1.5B 模型的 config（hidden=2048, layers=22, heads=32）等比缩小，得到一个"刚好能独立理解输入"的最小配置，就是这个甜点。

## 4. 共振场——协同的核心

### 4.1 场矩阵规格

```
场维度计算：
  D_field = hidden_size(标准) × N_标准神经元 / 压缩比

对于 30 个标准型神经元（hidden=768）、压缩比 6：
  D_field = 768 × 30 / 6 = 3840 → 取 4096

场矩阵：4096 × 4096 ≈ 16.8M 参数
```

**维度过小的后果**：神经元的写入向量互相覆盖，场退化为"平均值"→ 共振变成求和 → 1+1=1.5

**维度过大的后果**：空间太稀疏，神经元之间的条件化变弱 → 各自为政，无共振

**甜点**：能同时容纳 D/8 ≈ 512 个独立方向，但又密到 N_active / (D/8) > 0.15 时自然触发竞争。

### 4.2 循环共振机制

```
一轮循环 = 所有活跃神经元做一次前向传播

第 1 轮：
  - 所有 52 个神经元处理输入
  - 各自输出"写入向量"写到场
  - 场 = Sigma Vi （所有写入向量的叠加）

第 t 轮（t >= 2）：
  - 每个神经元读取当前场状态
  - 场状态作为"条件"注入神经元的中间层
  - 神经元输出 = f(input, 场状态)  ← 被场条件化
  - 更新场状态

条件化方式（在神经元内部）：
  神经元中间表示 h_mid = TransformerLayer(input)
  h_conditioned = h_mid + W_cond × 场状态  ← 残差注入
  继续前向传播
```

**为什么是乘性交互而非加性**：

传统 ensemble：output = w1·A + w2·B → 线性组合，永远 ≤ 二者之和

共振循环：output = 场(N轮)，其中：
```
场_t = 场_{t-1} + Sigma neuron_i(场_{t-1}, input)
```
因为每轮新输出都依赖上一轮的场状态（被条件化），信息的交互是乘性的：
- A 的输出依赖于 B 上轮的输出
- B 的输出依赖于 A 上轮的输出
- 两者在循环中互锁 → 涌现出新模式

如果 A 擅长中文、B 擅长代码，输入是"用中文解释这段 Python"：
- 第 1 轮：A 理解中文语义，B 理解代码结构
- 第 2 轮：A 读了 B 上轮的代码分析 → 调整中文表达；B 读了 A 的中文语义 → 重新理解代码意图
- 第 3 轮：两者理解互锁 → 场里出现了一个既懂代码又懂中文的"虚拟神经元"

## 5. 动态激活——场自己决定用多少神经元

### 5.1 共振度计算

```
对于每个神经元 i：
  Vi = 神经元输出的场写入向量（D 维）
  F  = 场当前状态（D 维，所有写入的叠加）

  score_i = (Vi · F) / (|Vi| × |F|)   ← 余弦相似度

  score_i in [-1, 1]
  → 1：神经元在"和场一起说话"
  → 0：神经元在自说自话
  → -1：神经元在和场对着干
```

### 5.2 拥堵度计算

```
拥堵度 = N_active / (D / 8)

D=4096 → 容量上限 = 512 个独立方向

N_active = 17 → 拥堵度 = 17/512 = 0.033  → 场很空
N_active = 80 → 拥堵度 = 80/512 = 0.156 → 场开始挤
```

### 5.3 动态阈值

```
阈值 T = 0.30 + 拥堵度 × 2.0

N_active = 5  → T = 0.30 + 0.01×2.0 = 0.32  → 容易进
N_active = 20 → T = 0.30 + 0.04×2.0 = 0.38  → 一般
N_active = 50 → T = 0.30 + 0.10×2.0 = 0.50  → 难进
N_active = 80 → T = 0.30 + 0.16×2.0 = 0.62  → 很难进
```

### 5.4 筛选流程

```
for 每一轮循环 t:
  for 每个神经元 i:
    score_i = cosine(Vi, F)
    if score_i > T:
      神经元 i 进入下一轮
    else:
      神经元 i 本轮被抑制

  更新 N_active → 重新计算 T

循环 3-5 轮后，自然收敛到稳定集合
```

### 5.5 实际效果演示

| 输入 | 过程 | 最终活跃 |
|------|------|----------|
| "你好" | 52→中文类~15→问候类~5 | **~5 个** |
| "Python 排序" | 52→代码+中文~20→Python 专属~6 | **~6 个** |
| "什么是机器学习" | 52→中文+英文+数学~18→解释类~8 | **~8 个** |
| "量子计算和 AI 关系" | 52→量子+AI+数学+哲学~14→两簇共振 | **~14 个** |

**没有人在"决定"**：阈值 T 是场的物理属性导出的。简单输入 → 场信号单一 → 大部分不共振 → 自然少。复杂输入 → 多个共振簇 → 自然多。

## 6. 三层自限机制——防止规模失控

### 6.1 层一：稀疏竞争（赢家通吃）

```
每次循环写入场后：
  1. 计算所有神经元的共振度
  2. 只保留 top-30% 的写入（其余衰减 alpha = 0.3）
  3. 被抑制的写入不是浪费——是多样性储备

效果：
  N=50 时 → 约 15 个实际主导
  N=200 时 → 约 60 个实际主导
  竞争让信号更清晰，而非更嘈杂
```

**类比大脑**：860 亿神经元同时只有 1-4% 在放电。侧向抑制不是 bug，是防止失控的核心机制。

### 6.2 层二：场维度动态生长

```
触发条件：
  N_active / (D/8) > 0.2 持续超过 100 轮

扩张流程：
  1. 记录当前场状态
  2. 创建新场（维度 × 1.5）
  3. 用旧场权重插值初始化新场
  4. 冷启动验证：新场共振度 > 旧场？
  OK 保留  FAIL 回滚

限制：
  最大 D = 16384（约 268M 场矩阵）
  超过此限制 → 触发层三
```

### 6.3 层三：集群自组织（分形分层）

```
当场维度达到上限、神经元数量持续增长时：

  1. 统计"共激活矩阵"：
     → 哪些神经元经常在同一轮被共振

  2. 聚类 → 形成"部落"
     每个部落内部有自己的子场

  3. 部落对外表现为一个"超级神经元"：
     → 输出 = 部落子场的最终状态
     → 写入 = 压缩后的场向量

  4. 上部场通信：
     → 超级神经元之间通过上级场共振
     → 通信是稀疏的、低频的

  5. 无限递归：
     神经元 → 部落 → 超级部落 → ...
     每层都是独立的共振场
```

**类比大脑**：6 层神经元形成微回路 → 微回路组成皮层柱 → 皮层柱之间长距连接。860 亿神经元不失控，靠的就是分层。大脑的容量上限不是 N 的线性函数，是对数函数。

## 7. 递归自进化——场的自然淘汰

### 7.1 进化周期

```
每日进化循环（在 4090D 上全自动）：

1. 评估阶段（运行中自动进行）
   - 场跟踪每个神经元的"长期共振度"
   - L_score_i = EMA(score_i, alpha=0.01)  ← 指数加权移动平均
   - L_score < 阈值 → 标记为"不共振"

2. 淘汰阶段
   - L_score 持续 7 天 < 0.15 → 移除
   - 该神经元的失败 case 被收集 → 生成新的训练数据需求

3. 培育阶段
   - 用失败领域数据训练一个紧凑型（15M）→ 加入场
   - 观察 24 小时的 L_score
   - 好 → 保留。不好 → 3 天内未达标则自动移除

4. 升级阶段
   - 紧凑型 L_score 持续 7 天 > 0.5 → 自动升级为标准型（50M）
   - 标准型 L_score 持续 30 天 > 0.6 → 自动升级为专家型（100M）
```

### 7.2 为什么不需要外部评估函数

传统模型进化需要人工评估（看 PPL、看生成质量、看 benchmark）。共振场不需要——因为**场的共振度本身就是评估函数**：

- 高共振度 = 这个神经元的输出在被其他神经元引用、条件化 → 有用
- 低共振度 = 它的写入每次都被人覆盖或忽略 → 没用

场本身就是一个"市场"：有用的神经元自然会收到反馈（共振），没用的自然被边缘化。

### 7.3 递归的闭环

```
培育 → 评估 → 淘汰 → 分析失败 → 生成新需求 → 培育新神经元

全程无需人工干预：
- 淘汰由场自动触发（共振度持续低下）
- 培育由淘汰触发（失败领域自动进入训练队列）
- 升级由场的接纳度触发（长期高共振 → 值得更大）
```

**递归的更深一层**：场的 W_cond（条件化权重）本身也在训练中持续更新。场是"学会和哪类神经元共振"的元知识——它记住的不只是内容，而是"什么样的组合能出好结果"。

## 8. 硬件需求分析

### 8.1 与单体模型的对比（1.5B 规模）

| | 单体 1.5B | 共振场（52 神经元 ≈1.5B） |
|---|---|---|
| **训练显存** | 14-18 GB | **3-6 GB**（一次只训一个） |
| **推理激活参数** | 1.5B（全部） | **~800M**（稀疏激活 ~16 个） |
| **推理显存** | 6-8 GB | **3-5 GB** |
| **推理计算量** | 1× | 3-5×（循环共振 3-5 轮） |
| **最低可运行显存** | ~8 GB | **2-4 GB** |
| **训练一个新能力** | 重新训练整个模型（天） | **训练一个神经元（分钟）** |
| **能力遗忘风险** | 每次更新可能影响全局 | **只影响替换的神经元** |

### 8.2 为什么显存低

- 训练：每次只需 50M 参数 + 场 17M = ~250MB 参数 + 梯度 ~500MB + 优化器 ~500MB + 激活 ~1GB ≈ **不到 3GB**
- 推理：稀疏激活 30%（~16 个神经元），场矩阵常驻
- 降级运行：显存紧张时减少激活神经元数量（5 个也能提供基本质量）

### 8.3 计算量 trade-off 的可接受性

3-5× 的计算量换来的是神经元间的乘性交互（1+1>2）。态极从未追求推理速度（1.4 tok/s 也接受），追求的是"活得更好"。这个 trade-off 与态极的哲学完全一致。

## 11. 神经元内部架构

### 11.1 统一结构

神经元不生成 token——它只做两件事：**理解输入，和场对话**。

```
                    input_ids
                        |
                 ┌──────▼──────┐
                 |  共享嵌入层   |  ← 256k×512, 全神经元共用（只加载一次）
                 |  524MB       |
                 └──────┬──────┘
                        |
                 ┌──────▼──────┐
                 |  Transformer |  ← 核心差异：层数/维度/头数决定神经元类型
                 |  N 层 Blocks |     RMSNorm + GQA + RoPE + SwiGLU（完全复用 lay.py）
                 └──┬───────┬──┘
                    |       |
           field_write    logits（预训练用，推理时不用）
         （写入共振场）     |
                    |       |
              归一化后写入场
```

### 11.2 三种规格参数

| | 紧凑型 | 标准型 | 专家型 |
|---|---|---|---|
| hidden_size | 512 | 768 | 1024 |
| num_layers | 6 | 10 | 14 |
| num_heads | 8 | 12 | 16 |
| num_kv_heads | 2 | 4 | 4 |
| intermediate_size | 1536 | 2304 | 3072 |
| Transformer 参数 | ~9M | ~28M | ~55M |
| field_write (→4096) | 2.1M | 3.1M | 4.2M |
| field_read_per_layer | 2.1M×6 | 3.1M×10 | 4.2M×14 |
| **总计** | **~24M** | **~59M** | **~118M** |

### 11.3 推理时的前向传播

```python
def forward(self, input_ids, field_state, round_num):
    h = SHARED_EMBEDDING(input_ids)  # 共享嵌入（全局唯一）

    for i, layer in enumerate(self.layers):
        h = h + layer.attn(layer.attn_norm(h))
        h = h + layer.ffn(layer.ffn_norm(h))

        # 场条件注入（第 2 轮开始生效）
        if field_state is not None and round_num > 1:
            h = h + self.field_read_layers[i](field_state)

    h = self.norm(h)
    v = self.field_write(h[:, -1, :])  # 写入场
    return v / (v.norm(dim=-1, keepdim=True) + 1e-8)  # L2 归一化
```

### 11.4 共享嵌入表——15M 神经元的关键

态极词表 256000 token。如果每个神经元独立维护嵌入表：

```
hidden=512  → 嵌入表 = 256000 × 512 × 4 bytes = 524 MB
hidden=768  → 嵌入表 = 256000 × 768 × 4 bytes = 786 MB
```

一个 15M 的紧凑型神经元光嵌入表就 500MB——完全不可行。

**所有神经元必须共享同一个嵌入层。** 嵌入层只加载一次，每个神经元的 24-118M 是纯 Transformer 层 + 场投影。共享嵌入在当前态极架构中天然存在（`backbone.embedding`），无需额外实现。

### 11.5 场读回机制

每层独立读取场状态：

```
层 0：field_state[4096] → field_read_0 → h[512]  残差注入
层 1：field_state[4096] → field_read_1 → h[512]  残差注入
...
层 N：field_state[4096] → field_read_N → h[512]  残差注入
```

浅层读到场后调整"注意力往哪儿放"，深层读到场后调整"最终方向往哪儿靠"。不共享 field_read 是因为场对浅层和深层的含义不同。

### 11.6 预训练阶段

推理时不需要 logits。但训练神经元时，需要它独立理解文本：

```
训练阶段 forward：
  1. 正常 Transformer 前向 → logits
  2. 标准交叉熵 loss（学语言）
  3. 最后 hidden 做 field_write（冻结保存，不参与训练 loss）
  4. field_read 在训练阶段随机初始化，在共振循环中微调
```

## 12. 大神经元会不会压制小神经元

### 12.1 归一化写入消除了规模差异

```
场写入前：Vi_norm = Vi / |Vi|  ← L2 归一化到单位长度

无论神经元是 15M 还是 100M，写进场的东西都是单位长度的向量。
大神经元不能"喊得更大声"——它只有一个单位长度的发言权。
```

### 12.2 共振度判的是方向，不是力量

```
score_i = cosine(Vi, F) = (Vi · F) / (|Vi| × |F|)

因为 Vi 和 F 都已归一化：score_i = Vi · F

这是一个纯方向度量，范围 [-1, 1]。和 Vi 的原始大小完全无关。
```

### 12.3 实际竞争场景

```
输入："用 Python 解释深度学习中梯度下降的原理"

紧凑型 A（15M，中文表述）：
  Vi 方向指向"解释类语言"→ score = 0.72 → 场认为它在正轨

专家型 B（100M，代码实现）：
  Vi 方向指向"代码输出"→ score = -0.15 → 场认为它偏了
```

同一个输入下，紧凑型的 score 远超专家型。不是因为"更强"，是因为**方向更对**。

切换输入为"写一个 PyTorch 训练循环"：

```
专家型 B：Vi 指向"代码实现"→ score = 0.85 → 主导
紧凑型 A：Vi 指向"解释类"→ score = 0.12 → 被筛掉
```

### 12.4 大神经元的真正优势

不是方向大小，是**方向的精准度**。专家型对还是不对更分明（方差小），多轮共振后更容易被场锁定。紧凑型对也是模糊对，错也模棱两可。稀疏竞争按 score 筛，不是按规模筛。

### 12.5 50 个小的会不会联合压制 2 个大的

不会。因为共振不是投票。归一化后，50 个平庸向量的叠加不一定在方向上胜过 2 个精准向量。谁的方差小、方向集中，谁就主导场的方向。

## 13. 超出限度的需求——共振簇

### 13.1 不是造更大的单体，是让神经元互锁

```
任务："详细分析 PyTorch 的 autograd 实现原理"

需要 5 个不同神经元协同：
  Python 代码专家（40M）+ 数学链式法则（60M）+
  中文技术写作（25M）+ 框架知识（50M）+ 代码生成（100M）

第 1 轮：各自独立理解
第 2 轮：互读场状态，重新条件化
第 3 轮：互锁完成 → 场里出现"虚拟神经元"
  → 等效参数 ≈ 40+60+25+50+100 = 275M
  → 且是按"最适合这个任务"的方式动态组合的
```

### 13.2 集群自组织提供无限上限

```
任务规模          → 共振簇大小          → 等效参数
简单问候            → 3-5 个紧凑型         → ~60M
代码解释            → 6-8 个（含专家）     → ~250M
完整系统设计        → 12-15 个跨领域       → ~600M
学科交叉研究        → 20+ 个多领域共振     → ~1B+
```

再往上触发层三（集群自组织），几个共振簇联合成"部落"，部落之间再通过上级场共振。等效参数理论上没有上限。

## 14. 算法可行性

### 14.1 有严格支撑的部分

| 机制 | 算法基础 | 验证状态 |
|------|----------|----------|
| 多专家协同 | MoE（Mixtral, Switch Transformer, DeepSeek-V3） | 已大规模生产验证 |
| 神经元结构 | Transformer（RMSNorm + GQA + RoPE + SwiGLU） | 标准架构，零新发明 |
| 场条件注入 | FiLM（Feature-wise Linear Modulation, 2018） | CV/NLP 领域大量验证 |
| 稀疏激活 | 赢家通吃竞争（Hopfield 网络收敛理论 + Switch Transformer top-k） | 理论完整 |
| 归一化写入 | L2 归一化（SimCLR, CLIP 对比学习标准做法） | 抗规模差异的标准方案 |
| 动态阈值 | 自适应退火（Hinton 2015 Distillation 论文中的温度调度） | 理论成熟 |

### 14.2 需要实验验证的部分

| 假设 | 风险 | 验证方法 |
|------|------|----------|
| 15M 神经元的场向量有语义质量 | 紧凑型输出可能是噪音 | 训练一个，线性分类器测区分度（<60%=失败，>75%=可用） |
| 循环共振 1+1>2 | 可能共振退化为平均 | 5 个神经元单测 vs 联合测 PPL |
| 进化周期收敛 | 可能震荡（培养→淘汰→培养） | 长期运行观察 EMA 稳定性 |
| 集群自组织 | 最遥远，缺乏实证 | 需要 50+ 神经元长期运行 |

### 14.3 什么情况下会失败

1. **场维度不够**：4096 维容纳不下 N 个独立方向 → 共振退化为平均 → 1+1=1.5
2. **紧凑型真的只是噪音**：15M 输出完全无区分度 → 场被污染
3. **进化震荡**：淘汰-培育循环不收敛 → 场状态持续波动
4. **场过度正则化**：W_cond 被少数优势神经元主导 → 失去接收新信号的能力

## 15. 生命周期与多模态

### 15.1 生命状态是场的内建分量

```
场状态 F 的维度结构：

F[:3500]    → 认知空间（神经元的"思考"在这里共振）
F[3500:3800]→ 生命空间（300 维，饥饿/好奇/疲劳/满足/无聊）
F[3800:4096]→ 元认知空间（296 维，"我在做什么""该不该信自己"）
```

生命状态影响共振方向：

```
好奇心高 → 场生命分量指向"探索"
  → 倾向于激活发散类神经元（哲学、创意）
  → 抑制收敛类神经元（代码规范、数学证明）

饥饿 → 倾向于激活效率类神经元 → 回答变短、更直接
```

### 15.2 进化周期 = 生命系统的长期记忆

```
场成功培育了一个代码神经元 → 生命维度"满足"上升，"好奇心"下降
第二天"好奇心"恢复 → 场重新启动探索轮
场淘汰了一个差神经元 → "反思"值上升
```

进化周期和生命周期是同一个东西。不是生命系统管理进化，是生命状态自然驱动进化。

### 15.3 多模态：每种模态是一类神经元

```
                 输入（文本 + 图像 + 音频）
                       |
          ┌────────────┼────────────┐
          |            |            |
     ┌────▼───┐   ┌───▼────┐   ┌───▼────┐
     | 文本    |   | 视觉    |   | 音频    |
     | 神经元  |   | 神经元  |   | 神经元  |
     | (50M)   |   |+ViT投影|   |+AudioEnc|
     └────┬───┘   └───┬────┘   └───┬────┘
          |            |            |
          └────────────┼────────────┘
                       |
                ┌──────▼──────┐
                |   共振场     |  ← 所有模态在同一空间共振
                └─────────────┘
```

视觉神经元：ViT + 投影器 → [512] → 和文本神经元完全相同的 Transformer 架构 → 写入共振场。

跨模态理解不是训练时固定的，是运行时从共振里涌现的。加新模态 = 加一个模态神经元，不重训任何已有的。

### 15.4 多模态 + 生命 + 进化的联动

```
视觉神经元发现用户照片里有很多食物
  → 场生命分量"饥饿"上升
  → 生命神经元检测到饥饿 + 视觉信号
  → 场激活推荐类神经元
  → 输出推荐餐厅

几周后推荐类 L_score 持续低 → 场淘汰它
  → 培育"健康饮食分析"神经元
  → 同样看到食物图片，输出变成营养分析
```

链条里没有硬编码规则。是视觉、生命状态、场共振、进化淘汰在数学上自然协同的结果。

## 16. 实现路线

### 16.1 现有代码复用

| 现有模块 | 操作 | 说明 |
|----------|------|------|
| `layers.py` | **不动** | RMSNorm、GQA、RoPE、SwiGLU、TransformerBlock——所有基础算法零改动 |
| `architecture.py` / `pretrain.py` | **不动** | TaijiBackbone 仍是神经元内部结构，只是 config 缩放 |
| `loader.py` | **不动** | load_model 直接用于加载各神经元 |
| `tokenizer_native_v2/` | **不动** | 嵌入表共享，所有神经元同一 tokenizer |
| `config.py` | **加配置** | 三套神经元默认 config |

### 16.2 需新增

| 文件 | 行数 | 功能 |
|------|------|------|
| `taiji/resonance/field.py` | ~300 | 共振场核心：归一化写入、赢家通吃、场自更新 |
| `taiji/resonance/neuron.py` | ~150 | 封装 TaijiBackbone + field_write + field_read |
| `taiji/resonance/ensemble.py` | ~200 | 共振循环调度器：多轮循环 + 动态阈值筛选 |
| `taiji/config.py` | +30 行 | 紧凑型/标准型/专家型配置 |

总计不到 **700 行新代码**。所有基础算法一行不改。

### 16.3 最低验证实验（4090D 一周可完成）

```bash
# Step 1：训练 5 个不同领域的 50M 神经元
# 中文对话、英文技术、代码生成、数学推理、通用知识

# Step 2：实现最小共振场（Python 脚本，200 行）
# 场维度 2048，归一化写入，3 轮循环共振

# Step 3：对比实验
# A 组：5 个神经元单独推理 PPL
# B 组：5 个神经元在共振场中联合推理 PPL
# C 组：1 个 250M 单体（5×50M）的 PPL

# 成功标准：
# B < A_best → 共振成立
# B < C_avg → 1+1>2 成立
```
