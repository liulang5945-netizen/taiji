# 态极共振场架构（Resonance Field Architecture）

> **核心思想**：态极不是一个大的 Transformer 模型，而是由若干独立的小模型神经元组成。
> 所有神经元通过一个共享的"共振场"协同工作，从中涌现出统一的智能。
> 自我进化通过增加/替换神经元实现，递归通过场自然淘汰不共振的神经元实现。

---

## 1. 为什么不是 MoE

MoE（Mixture of Experts）的核心是"分而治之"——路由器选择 top-k 个专家来处理输入。

| | MoE 路由 | 共振场 |
|---|---|---|
| 神经元激活 | 路由器决定选谁 | **所有神经元同时激活，场自然筛选** |
| 协同方式 | 加权投票（线性） | **循环共振（乘性交互）** |
| 1+1 | 最多等于 2 | **可以大于 2** |
| 规模增长 | 增加专家 → 重新训练路由器 | 增加神经元 → 即插即用 |
| 进化淘汰 | 需要外部评估 | **场自然排斥不共振的神经元** |

共振场的核心区别：不是"选几个用"，而是"所有人同时说话，场决定听谁的"。

## 2. 整体架构

```
                        输入
                          |
                ┌─────────┼─────────┐
                |         |         |
           ┌────▼───┐ ┌───▼────┐ ┌───▼────┐
           |神经元 A | |神经元 B| |神经元 C|   ← 全量激活（所有 52 个）
           | 50M 参数| | 50M 参数| | 50M 参数|
           | hidden | | hidden | | hidden |
           └────┬───┘ └───┬────┘ └───┬────┘
                |         |         |
                └─────────┼─────────┘
                          |
                   ┌──────▼──────┐
                   |   共振场     |  ← D=4096 维共享矩阵
                   |  (Field)    |     所有神经元写入
                   └──────┬──────┘     所有神经元读回
                          |
                ┌─────────┼─────────┐
                | 读回     | 读回     | 读回
           ┌────▼───┐ ┌───▼────┐ ┌───▼────┐
           |神经元 A | |神经元 B| |神经元 C|   ← 第二轮循环
           | 条件化  | | 条件化 | | 条件化 |   被场状态影响后的重新推理
           └────┬───┘ └───┬────┘ └───┬────┘
                |         |         |
                └─────────┼─────────┘
                          |
                   ┌──────▼──────┐
                   |  更新共振场  |
                   └──────┬──────┘
                          |
                   循环 3-5 轮
                          |
                   ┌──────▼──────┐
                   |   输出       |
                   └─────────────┘
```

**关键特征**：
- 所有神经元同时激活（不存在"选谁不选谁"）
- 每轮循环：神经元写入场 → 从场读回 → 重新计算
- 场的介入让神经元的输出彼此"条件化"
- 多轮循环后，场的干涉模式自然涌现为输出

## 3. 神经元规格设计

神经元不是一样大的。三种规格，三种角色：

### 3.1 规格表

| | 紧凑型 | 标准型 | 专家型 |
|---|---|---|---|
| **参数量** | 15-25M | 40-60M | 80-120M |
| **hidden_size** | 512 | 768 | 1024 |
| **num_layers** | 6 | 10 | 14 |
| **num_heads** | 8 | 12 | 16 |
| **num_kv_heads** | 2 | 4 | 4 |
| **intermediate_size** | 1536 | 2304 | 3072 |
| **4090D 训练时间** | 3-5 分钟 | 10-15 分钟 | 25-35 分钟 |
| **角色** | 快速试错、探索新领域 | 主力神经元 | 复杂推理、代码、数学 |
| **生命周期** | 短（容易淘汰） | 中 | 长（通过进化验证） |
| **推荐数量** | 20-40 个 | 8-15 个 | 2-4 个 |

### 3.2 设计原理

**太小不行**：<5M 的模型连"理解一句话"都做不到，加到共振场里只是噪音。

**太大不行**：>200M 训练慢、进化慢，丧失了即插即用的灵活性。而且过大的神经元会对场形成"主导效应"，压制小神经元，破坏共振。

**异质化的意义**：用态极现有 1.5B 模型的 config（hidden=2048, layers=22, heads=32）等比缩小，得到一个"刚好能独立理解输入"的最小配置，就是这个甜点。

## 4. 共振场——协同的核心

### 4.1 场矩阵规格

```
场维度计算：
  D_field = hidden_size(标准) × N_标准神经元 / 压缩比

对于 30 个标准型神经元（hidden=768）、压缩比 6：
  D_field = 768 × 30 / 6 = 3840 → 取 4096

场矩阵：4096 × 4096 ≈ 16.8M 参数
```

**维度过小的后果**：神经元的写入向量互相覆盖，场退化为"平均值"→ 共振变成求和 → 1+1=1.5

**维度过大的后果**：空间太稀疏，神经元之间的条件化变弱 → 各自为政，无共振

**甜点**：能同时容纳 D/8 ≈ 512 个独立方向，但又密到 N_active / (D/8) > 0.15 时自然触发竞争。

### 4.2 循环共振机制

```
一轮循环 = 所有活跃神经元做一次前向传播

第 1 轮：
  - 所有 52 个神经元处理输入
  - 各自输出"写入向量"写到场
  - 场 = Sigma Vi （所有写入向量的叠加）

第 t 轮（t >= 2）：
  - 每个神经元读取当前场状态
  - 场状态作为"条件"注入神经元的中间层
  - 神经元输出 = f(input, 场状态)  ← 被场条件化
  - 更新场状态

条件化方式（在神经元内部）：
  神经元中间表示 h_mid = TransformerLayer(input)
  h_conditioned = h_mid + W_cond × 场状态  ← 残差注入
  继续前向传播
```

**为什么是乘性交互而非加性**：

传统 ensemble：output = w1·A + w2·B → 线性组合，永远 ≤ 二者之和

共振循环：output = 场(N轮)，其中：
```
场_t = 场_{t-1} + Sigma neuron_i(场_{t-1}, input)
```
因为每轮新输出都依赖上一轮的场状态（被条件化），信息的交互是乘性的：
- A 的输出依赖于 B 上轮的输出
- B 的输出依赖于 A 上轮的输出
- 两者在循环中互锁 → 涌现出新模式

如果 A 擅长中文、B 擅长代码，输入是"用中文解释这段 Python"：
- 第 1 轮：A 理解中文语义，B 理解代码结构
- 第 2 轮：A 读了 B 上轮的代码分析 → 调整中文表达；B 读了 A 的中文语义 → 重新理解代码意图
- 第 3 轮：两者理解互锁 → 场里出现了一个既懂代码又懂中文的"虚拟神经元"

## 5. 动态激活——场自己决定用多少神经元

### 5.1 共振度计算

```
对于每个神经元 i：
  Vi = 神经元输出的场写入向量（D 维）
  F  = 场当前状态（D 维，所有写入的叠加）

  score_i = (Vi · F) / (|Vi| × |F|)   ← 余弦相似度

  score_i in [-1, 1]
  → 1：神经元在"和场一起说话"
  → 0：神经元在自说自话
  → -1：神经元在和场对着干
```

### 5.2 拥堵度计算

```
拥堵度 = N_active / (D / 8)

D=4096 → 容量上限 = 512 个独立方向

N_active = 17 → 拥堵度 = 17/512 = 0.033  → 场很空
N_active = 80 → 拥堵度 = 80/512 = 0.156 → 场开始挤
```

### 5.3 动态阈值

```
阈值 T = 0.30 + 拥堵度 × 2.0

N_active = 5  → T = 0.30 + 0.01×2.0 = 0.32  → 容易进
N_active = 20 → T = 0.30 + 0.04×2.0 = 0.38  → 一般
N_active = 50 → T = 0.30 + 0.10×2.0 = 0.50  → 难进
N_active = 80 → T = 0.30 + 0.16×2.0 = 0.62  → 很难进
```

### 5.4 筛选流程

```
for 每一轮循环 t:
  for 每个神经元 i:
    score_i = cosine(Vi, F)
    if score_i > T:
      神经元 i 进入下一轮
    else:
      神经元 i 本轮被抑制

  更新 N_active → 重新计算 T

循环 3-5 轮后，自然收敛到稳定集合
```

### 5.5 实际效果演示

| 输入 | 过程 | 最终活跃 |
|------|------|----------|
| "你好" | 52→中文类~15→问候类~5 | **~5 个** |
| "Python 排序" | 52→代码+中文~20→Python 专属~6 | **~6 个** |
| "什么是机器学习" | 52→中文+英文+数学~18→解释类~8 | **~8 个** |
| "量子计算和 AI 关系" | 52→量子+AI+数学+哲学~14→两簇共振 | **~14 个** |

**没有人在"决定"**：阈值 T 是场的物理属性导出的。简单输入 → 场信号单一 → 大部分不共振 → 自然少。复杂输入 → 多个共振簇 → 自然多。

## 6. 三层自限机制——防止规模失控

### 6.1 层一：稀疏竞争（赢家通吃）

```
每次循环写入场后：
  1. 计算所有神经元的共振度
  2. 只保留 top-30% 的写入（其余衰减 alpha = 0.3）
  3. 被抑制的写入不是浪费——是多样性储备

效果：
  N=50 时 → 约 15 个实际主导
  N=200 时 → 约 60 个实际主导
  竞争让信号更清晰，而非更嘈杂
```

**类比大脑**：860 亿神经元同时只有 1-4% 在放电。侧向抑制不是 bug，是防止失控的核心机制。

### 6.2 层二：场维度动态生长

```
触发条件：
  N_active / (D/8) > 0.2 持续超过 100 轮

扩张流程：
  1. 记录当前场状态
  2. 创建新场（维度 × 1.5）
  3. 用旧场权重插值初始化新场
  4. 冷启动验证：新场共振度 > 旧场？
  OK 保留  FAIL 回滚

限制：
  最大 D = 16384（约 268M 场矩阵）
  超过此限制 → 触发层三
```

### 6.3 层三：集群自组织（分形分层）

```
当场维度达到上限、神经元数量持续增长时：

  1. 统计"共激活矩阵"：
     → 哪些神经元经常在同一轮被共振

  2. 聚类 → 形成"部落"
     每个部落内部有自己的子场

  3. 部落对外表现为一个"超级神经元"：
     → 输出 = 部落子场的最终状态
     → 写入 = 压缩后的场向量

  4. 上部场通信：
     → 超级神经元之间通过上级场共振
     → 通信是稀疏的、低频的

  5. 无限递归：
     神经元 → 部落 → 超级部落 → ...
     每层都是独立的共振场
```

**类比大脑**：6 层神经元形成微回路 → 微回路组成皮层柱 → 皮层柱之间长距连接。860 亿神经元不失控，靠的就是分层。大脑的容量上限不是 N 的线性函数，是对数函数。

## 7. 递归自进化——场的自然淘汰

### 7.1 进化周期

```
每日进化循环（在 4090D 上全自动）：

1. 评估阶段（运行中自动进行）
   - 场跟踪每个神经元的"长期共振度"
   - L_score_i = EMA(score_i, alpha=0.01)  ← 指数加权移动平均
   - L_score < 阈值 → 标记为"不共振"

2. 淘汰阶段
   - L_score 持续 7 天 < 0.15 → 移除
   - 该神经元的失败 case 被收集 → 生成新的训练数据需求

3. 培育阶段
   - 用失败领域数据训练一个紧凑型（15M）→ 加入场
   - 观察 24 小时的 L_score
   - 好 → 保留。不好 → 3 天内未达标则自动移除

4. 升级阶段
   - 紧凑型 L_score 持续 7 天 > 0.5 → 自动升级为标准型（50M）
   - 标准型 L_score 持续 30 天 > 0.6 → 自动升级为专家型（100M）
```

### 7.2 为什么不需要外部评估函数

传统模型进化需要人工评估（看 PPL、看生成质量、看 benchmark）。共振场不需要——因为**场的共振度本身就是评估函数**：

- 高共振度 = 这个神经元的输出在被其他神经元引用、条件化 → 有用
- 低共振度 = 它的写入每次都被人覆盖或忽略 → 没用

场本身就是一个"市场"：有用的神经元自然会收到反馈（共振），没用的自然被边缘化。

### 7.3 递归的闭环

```
培育 → 评估 → 淘汰 → 分析失败 → 生成新需求 → 培育新神经元

全程无需人工干预：
- 淘汰由场自动触发（共振度持续低下）
- 培育由淘汰触发（失败领域自动进入训练队列）
- 升级由场的接纳度触发（长期高共振 → 值得更大）
```

**递归的更深一层**：场的 W_cond（条件化权重）本身也在训练中持续更新。场是"学会和哪类神经元共振"的元知识——它记住的不只是内容，而是"什么样的组合能出好结果"。

## 8. 硬件需求分析

### 8.1 与单体模型的对比（1.5B 规模）

| | 单体 1.5B | 共振场（52 神经元 ≈1.5B） |
|---|---|---|
| **训练显存** | 14-18 GB | **3-6 GB**（一次只训一个） |
| **推理激活参数** | 1.5B（全部） | **~800M**（稀疏激活 ~16 个） |
| **推理显存** | 6-8 GB | **3-5 GB** |
| **推理计算量** | 1× | 3-5×（循环共振 3-5 轮） |
| **最低可运行显存** | ~8 GB | **2-4 GB** |
| **训练一个新能力** | 重新训练整个模型（天） | **训练一个神经元（分钟）** |
| **能力遗忘风险** | 每次更新可能影响全局 | **只影响替换的神经元** |

### 8.2 为什么显存低

- 训练：每次只需 50M 参数 + 场 17M = ~250MB 参数 + 梯度 ~500MB + 优化器 ~500MB + 激活 ~1GB ≈ **不到 3GB**
- 推理：稀疏激活 30%（~16 个神经元），场矩阵常驻
- 降级运行：显存紧张时减少激活神经元数量（5 个也能提供基本质量）

### 8.3 计算量 trade-off 的可接受性

3-5× 的计算量换来的是神经元间的乘性交互（1+1>2）。态极从未追求推理速度（1.4 tok/s 也接受），追求的是"活得更好"。这个 trade-off 与态极的哲学完全一致。

## 9. 规模推演

### 第一代（从零启动）
```
配置：10 个紧凑型（15M）、场 D=1536（约 2.3M）
总参数：152M
训练时间：每神经元 3 分钟 × 10 = 30 分钟
品质：相当于 ~200M 单体模型
```

### 第十代（一个月后）
```
配置：25 紧凑（15M）+ 8 标准（50M）+ 2 专家（100M）、场 D=4096（约 16.8M）
总参数：992M
品质：远超 1B 单体（因为结构是进化出来的，不是随机初始化的）
```

### 长期（三个月后）
```
配置：通过集群自组织形成 3-5 个部落，对外等效 3-5B
实际参数总量可能超过 5B，但推理时只激活 ~1.5B
达到传统训练方式在 4090D 上永远达不到的规模
```

## 10. 与态极现有基础设施的兼容性

| 现有模块 | 在新架构中的位置 |
|----------|-------------------|
| pretrain.py / 训练管线 | 单神经元训练器（无需修改） |
| taiji/architecture.py | 神经元的内部结构（三种规格的 config） |
| taiji/tokenizer_native_v2/ | 所有神经元共享同一 tokenizer |
| 数据配比引擎 | 决定新神经元的训练数据方向 |
| 生命系统 | 进化的调度器（何时评估、何时培育） |
| Agent 头（五头 backbone） | 每个神经元内部可选的小头，或整体对外的大头 |

**关键优势**：不需要推倒重来。现有训练的每个 checkpoint 都可以作为标准型神经元的初始权重。场矩阵和循环机制是新增的上层模块。
