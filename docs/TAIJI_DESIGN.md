# 态极设计理念

## 核心理念

态极是**生命体**，不是 AI 工具。

### 传统 AI 应用 vs 态极

| 传统 AI 应用 | 态极 |
|-------------|------|
| 可以选择不同模型 | 有唯一的核心模型（大脑） |
| 展示技术细节 | 展示生命状态 |
| 工具调用 | 行动能力 |
| API 接口 | 直接交流 |

### 生命体特征

1. **唯一性** — 态极有唯一的核心模型，就像人类有唯一的大脑
2. **整体性** — 态极是一个完整的个体，不是可组合的模块
3. **生命状态** — 态极有健康、行动、代谢、感知等状态
4. **互动性** — 可以喂养、睡眠、玩耍，而不是调用 API

## 界面设计

### 态极形象

```
    😊
   态极
```

- 显示态极的表情（醒着/睡着）
- 显示态极的状态文本

### 生命状态

```
身体状态: 健康
行动能力: 可用
代谢系统: 正常
感知能力: 灵敏
```

### 互动按钮

```
[🍎 喂养] [😴 睡眠] [🎮 玩耍]
```

### 对话区域

```
👤 用户消息

😊 态极回复
```

## 技术实现

### 前端 (Vue)

- 不显示技术细节（模型名称、工具列表等）
- 只显示生命状态和互动界面
- 使用表情和动画表达态极的状态

### 后端 (Rust)

- 调用 Python 核心接口
- 只暴露生命相关的命令：
  - `get_life_status` — 获取生命状态
  - `chat` — 与态极对话
  - `feed` — 喂养态极
  - `sleep` — 让态极睡眠
  - `play` — 让态极玩耍

### Python 核心

- 提供 TaijiCore 类
- 管理态极的生命状态
- 不暴露技术细节

## 未来扩展

1. **情感系统** — 态极可以表达情感
2. **成长系统** — 态极可以学习和成长
3. **记忆系统** — 态极可以记住过去的互动
4. **个性系统** — 态极可以发展独特的个性
